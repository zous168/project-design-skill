# CRM 系统架构设计

> 全闭环产出 · 对应 skill 文档 **03**。上游:[01 需求规格](01-需求规格.md)、[02 领域分析](02-领域分析.md)、[08 非功能性需求](08-非功能性需求.md)。下游:[07 异步处理](07-异步处理.md)、[09 技术选型](09-技术选型.md)、[10 工程目录结构](10-工程目录结构.md)。

## 1. 架构目标与驱动(NFR)
- 私有化/集中部署灵活(部署驱动)→ **模块化单体**,单一可部署单元、模块清晰。
- 销售数据唯一源、规则集中(一致性驱动)→ 域模块边界 + 共享契约。
- 与 Hermes/客户全景/CSM 多系统集成(可替换/降级驱动)→ **集成适配器隔离**。
- 进线查重/话术检索低延迟(性能驱动,P95≤200ms)→ cfg 可缓存、读写分库。

## 2. 上下文视图(三层中台中的位置)
```
通道层(官网/微信/企微/抖音 Automan/站点) → 智能层 Hermes(唯一生成式出口)
                                                  │ 工具插件 + Webhook
                                                  ▼
业务数据层:  [客户全景(主数据)] ── [CRM(销售域·本系统)] ── [CSM(售后)] ── [订单]
                         经 customerId / orderId 协同
```
CRM 不直接感知通道;经 Hermes 调用本系统工具插件。

## 3. 子系统分解(Phase 0 确认)
| 子系统 | 职责 | 用户 | 共享 |
|--------|------|------|------|
| **坐席工作台(Web)** | 工作台/线索/商机/跟进 | R1/R2 | 同一后端 + data/cfg 库 |
| **管理后台(Admin)** | SOP/话术/角色/渠道分配规则/报表配置 | R3 | 同一后端 + cfg 库 |
| **开放 API(Hermes)** | 建线索/回写跟进/查 SOP/检索话术/赢单回调(无界面) | R4 | 同一后端,REST + API Key |

> 三子系统共用后端(模块化单体),前端两个 SPA(工作台/后台)+ 一组开放 API;详见各自功能树([04](04-功能树.md))与接口([06](06-接口设计.md))。

## 4. 逻辑视图(后端模块,模块化单体)
`modules/`: lead · opportunity · follow-up · playbook · sop · analytics;
`integration/`: hermes · customer360 · csm(适配器);`common/`(鉴权/幂等/错误模型)。
域模块间不互相 import 内部,经 service 接口/领域事件协作(详见 [10 工程目录结构](10-工程目录结构.md))。

## 5. 关键运行时流程
**A. 留资进线(查重+分配)**
```
渠道→Hermes: identify(取 customerId) → CRM POST /api/v1/leads(X-Request-Id)
CRM: 查重(BR-1) → [命中]返回原 leadId  /  [新]建 Lead + 自动分配(BR) → 202/201
```
**B. 赢单回写(BR-3)**
```
坐席→CRM: PATCH /opportunities/{id}/stage = 赢单
CRM(事务) → 发"OpportunityWon"事件 → 异步(07):
   ├─ customer360.behaviorSync(成交)   ├─ csm.notifyWon(开通售后)   └─ hermes.callback(售后触达)
   (各幂等,失败进 DLQ;记录 → logs)
```

## 6. 部署视图
- 单可部署后端(API 进程)+ **独立 worker 进程**(异步/调度,见 [07](07-异步处理.md));Redis(队列+缓存)、主库(data/cfg)、日志/统计库(logs/stats)。
- 形态:本地私有化 与 服务器集中部署同构;能力与部署位置无关。Hermes 不可用时,开放 API 可降级为"仅记录留资"。

## 7. 横切关注点
鉴权(工作台 JWT / 开放 API API Key)· 统一错误模型 + 幂等(X-Request-Id)· 全量操作审计→ logs · 配置中心(cfg)· i18n/主题为前端横切(见 [08 NFR](08-非功能性需求.md))。

## 8. 架构决策记录(ADR)
| ADR | 决策 | 备选 | 理由 |
|-----|------|------|------|
| ADR-1 | **模块化单体** | 微服务 / 单体 | 契合私有化/集中部署、运维轻;模块边界即未来微服务切分线 |
| ADR-2 | API **统一 REST**(`/api/v1`) | GraphQL/gRPC | 中后台 + 集成场景成熟、可 Swagger 化;事件走异步(07) |
| ADR-3 | 数据**分 cfg/data/logs/stats 四库** | 单库 | 配置可缓存/灰度、日志只追加可归档、统计可重算(见 [05](05-数据设计.md)) |
| ADR-4 | 赢单等副作用走**异步队列 + 适配器** | 同步直连 | 解耦外部系统、可重试/降级(见 [07](07-异步处理.md)) |
