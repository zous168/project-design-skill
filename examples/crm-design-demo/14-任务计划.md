# CRM 任务计划与交付流程

> 全闭环产出 · 对应 skill 文档 **14**。任务**从 [04 功能树](04-功能树.md) 派生**、按 [13 路线图](13-路线图.md) 排序;供开发 Agent/团队按闭环执行。

## 1. 切片规则
每任务独立验收、独立跑测;禁止一次吞整模块/系统。大任务(>5 文件 / 跨≥2 目录栈 / "整模块·全接口·整页")**先拆子任务清单**。粒度:**API**=一资源或对称端点组;**页面**=一条用户路径;**迁移**=一批相关表(API 同切片或下一切片)。

## 2. M1(MVP)切片清单(摘录,串行门禁)
- [ ] S1 线索资源 API(`/api/v1/leads` CRUD + 查重 FR-L1/L2)
- [ ] S2 线索分配 API + 规则(FR-L4)
- [ ] S3 线索管理页(列表/筛选/录入/分配 + 三态)
- [ ] S4 商机资源 API + 阶段推进(FR-O1/O2)
- [ ] S5 跟进 API + Hermes 回写(FR-F1/F3)
- [ ] S6 工作台页(待办队列 + 商机详情 + 客户全景只读)
- [ ] S7 开放 API:建线索/回写(FR-I1,API Key + 幂等)

## 3. 任务卡(示例:S1)
```
【名称】线索资源 API + 查重
【父/序号】M1 / S1
【文档锚点】FR-L1, FR-L2;[04 功能树] C1/A2;[06 接口] POST /api/v1/leads, /leads:dedup
【范围:含】建/查/列线索、查重(手机号+渠道ID+有效期)、来源标记
【范围:不含】分配规则(S2)、前端页面(S3)
【完成标准】重复进线返回原 leadId 且 deduped=true;字段对齐 [05 数据] lead 表;OpenAPI 出现该端点
【测试用例】同号重复→同id;新号→新建;非法入参→400;幂等(X-Request-Id)
【测试命令】<模块测试约定>(如 pnpm --filter server test lead)
【UI 验证】无(后端切片)
【UI 样式】无
```

## 4. 门禁(串行交付)
S1 全绿 → S2 → …;只读探查可并行,交付串行;每切片走下面闭环;未全绿禁止下一切片;进度在汇报中勾选。

## 5. 每切片闭环 / DoD（①–⑨）
① Read 文档链+现有实现(记版本)· ② Plan(无锚点先文档化;大任务出子清单;改文件≤5)· ③ Implement(活文档+模块约定,最小 diff)· ④ Test(自查测试法+用例+命令;UI→浏览器点验+样式 pre-flight)· ⑤ Triage(实现/测试/文档三分类,禁 skip)· ⑥ Fix(自行重启依赖,重跑至全绿,不甩用户)· ⑦ SelfCheck · ⑧ Sync(需求变更同步活文档+bump 版本)· ⑨ Report(子任务简报→总汇报)。

**失败三分类:** 实现 bug→改码;测试 bug→改测试/配置;文档/用例 bug→先改文档或任务卡再改码。**未重跑测试不得标完成。**
**Agent 默认:** 不 commit/push(除非要求);不扩 scope;最小 diff。
