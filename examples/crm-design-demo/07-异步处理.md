# CRM 异步 / 后台处理设计

> 全闭环产出 · 对应 skill 文档 **07**。上游:[03 架构](03-系统架构.md)、[04 功能树](04-功能树.md)、[05 数据设计](05-数据设计.md)。覆盖 API([06](06-接口设计.md))之外的执行:job worker / 异步任务 / 调度任务。

## 1. 任务清单

## 2. 分类与执行模型
- **异步任务**:操作的副作用,API 快速返回(写操作 → 入队 → 202/即时返回),如赢单回写、触达回写。
- **Job worker**:队列消费/批处理(导入、RAG 重建),独立 worker 进程。
- **调度/cron**:周期任务(rollup、唤醒、SLA 提醒、归档清理、话术到期)。

## 3. 基础设施(与 [09 技术选型](09-技术选型.md) 一致)
- **BullMQ + Redis**(NestJS):队列、延迟队列、可重复(repeatable)调度;**独立 worker 进程**与 API 进程分离部署(见 [03 部署视图](03-系统架构.md))。
- 私有化部署内置 Redis;调度既可 BullMQ repeatable,也可 k8s CronJob(集中部署)。

## 4. 可靠性语义
- **至少一次** + 业务幂等(requestId / 事件 id / 当日键);失败**指数退避重试**,超阈进 **DLQ** 并告警。
- **调度防重跑**:HA/多实例下用 **Redis 分布式锁(或 BullMQ repeatable 单实例语义)** 保证每个 cron 窗口只执行一次。
- 背压:队列积压超阈告警 + 限流入队。

## 5. 触发映射(功能树/事件 → 任务 → 落库)
- FR-O3 赢单 → `OpportunityWon` → 三回写任务 → 记录 `logs.api_log`/事件日志。
- FR-F3 Hermes 触达 → 回写任务 → `data.follow_up` + `logs.hermes_touch_log`(traceId)。
- FR-A1/A2 统计 → rollup 调度 → `stats.funnel_daily`/`sales_perf_daily`。
- FR-S 唤醒/SLA → 调度 → 入队提醒,记 logs。

## 6. 可观测与运维
- 指标:队列深度、成功/失败数、处理延迟、DLQ 大小(→ 监控)。
- 告警:DLQ 非空、rollup 失败、积压超阈。
- Runbook:DLQ 重放、调度补算;job 记录保留(logs 90 天热 + 归档)。
