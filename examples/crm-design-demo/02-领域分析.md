# CRM 业务与领域分析

> 全闭环产出 · 对应 skill 文档 **02**。上游:[01 需求规格](01-需求规格.md)。下游复用本文实体命名:[05 数据设计](05-数据设计.md)、[06 接口设计](06-接口设计.md)。

## 1. 领域概览
销售域围绕"把线索培育成成交"展开。CRM 是**销售数据唯一权威源 + 规则管控中心**:沉淀线索/商机/跟进、定义阶段与 SOP、管控报价折扣、提供人工兜底;不做生成式应答(归 Hermes)、不存客户主数据(归客户全景)、不做售后(归 CSM)。

## 2. 领域模型(实体与关系)
```
客户全景[customerId] 1───* Lead 线索 ──converts──> Opportunity 商机 1───* FollowUp 跟进
                                          │                       └─ 关联 traceId(Hermes 触达)
Opportunity *──> Quote 报价 ──>(<9折)──> SpecialPriceApproval 特价审批
Playbook 话术 *──RAG──> Hermes        SopTemplate SOP *──查询──> Hermes
User 用户 *──belongs──> Role 角色      Channel 渠道 / AssignRule 分配规则(cfg)
```
- 核心实体:**Lead, Opportunity, FollowUp, Quote, SpecialPriceApproval, Playbook, SopTemplate, User, Role, Channel, AssignRule**。客户为**外部引用**(仅 `customerId`)。
- 命名为 ubiquitous language,下游(数据/接口/功能树)逐字复用。

## 3. 核心业务流程
1. **留资承接:** 渠道留资 → Hermes 调 `identify` 取 customerId → 建 Lead(查重)→ 自动分配。
2. **培育转化:** Lead 跟进 → 满足条件转 Opportunity → 按阶段推进 → 报价(折扣校验)→ 赢单/输单。
3. **赢单闭环:** 赢单 → 回写客户全景(成交) + 通知 CSM(开通售后) + 回调 Hermes。
4. **AI 协同:** Hermes 按 SOP 场景编码取节点/权限/话术,在规则内触达,回写跟进(traceId);越权(如折扣超限)挂起转人工。

## 4. 业务规则(BR,可测)
| ID | 规则 |
|----|------|
| BR-1 | 同手机号/渠道 ID 在有效周期内不重复建线索,命中返回原 `leadId` |
| BR-2 | 报价折扣 ≥9 折坐席自助;<9 折必经主管审批(SpecialPriceApproval) |
| BR-3 | 赢单必触发:客户全景回写 + CSM 通知 + Hermes 回调(三者幂等) |
| BR-4 | 话术分域:销售素材仅本域;售后 FAQ 归 CSM;Hermes 按 Agent 类型路由 |
| BR-5 | Hermes 不得突破 SOP 返回的权限边界(如未授权折扣) |
| BR-6 | 客户主数据只读引用,成交/跟进等行为回写客户全景 |

## 5. 关键状态机
- **Lead.status:** 新建 → 跟进中 → 已转化 / 已关闭 / 无效(变更留痕 → logs)。
- **Opportunity.stage:**(可配置)意向确认 → 方案报价 → 谈判 → 赢单 / 输单。赢单→触发 BR-3。
- **SpecialPriceApproval.status:** 待审 → 通过 / 驳回。

## 6. 干系人与关注点
坐席(效率/兜底)· 主管(管控/团队转化)· 运营(标准化/数据)· Hermes(规则边界)· 客户全景&CSM&订单(平级域,经 id 协同)。
