# CRM 数据设计（cfg / data / logs / stats 四库）

> 试跑产出（对应 skill 文档 **05 — Data Design**）。按四个逻辑库划分；**跨库仅以 id 引用，不做跨库外键**；客户主数据归客户全景域，本系统仅存 `customerId`。表结构以**模型表**描述，不含 DDL。

## 四库职责与归属

| 库 | 职责 | 主要实体 |
|----|------|----------|
| **cfg** | 配置 / 元数据 / 规则 / 字典 | role, permission, user, channel, assign_rule, sop_template, sop_version, playbook, playbook_version, stage_config |
| **data** | 核心业务实体与事务 | lead, opportunity, follow_up, quote, special_price_approval |
| **logs** | 审计 / 操作 / 事件日志 | op_log, api_log, hermes_touch_log, stage_change_log |
| **stats** | 聚合 / 报表 / 指标 | funnel_daily, sales_perf_daily, playbook_hit_daily |

> 选择依据：配置类低频写、需版本与灰度（cfg 独立便于缓存与发布）；业务事务高频读写（data）；日志只追加、量大、可独立归档（logs）；统计为派生只读、可重算（stats）。

## data 库 · 关键实体

### lead（线索）

|------|------|:--:|------|
| id | bigint(PK) | ✓ | |

| phone / channel_uid | varchar | ✓ | 查重键之一 |

### opportunity（商机）
| 字段 | 类型 | 必填 | 说明 |

| amount_expected | decimal | | 预计金额 |

### follow_up（跟进）/ special_price_approval（特价审批）
- follow_up：id, opportunity_id, channel(电话/企微/面谈), summary, next_at, actor(user/hermes), trace_id(→logs 关联), created_at。
- special_price_approval：id, opportunity_id, discount, reason, applicant_id, approver_id, status(待审/通过/驳回)，体现 §4.2 折扣权限。

## cfg 库 · 要点
- **role / permission / user**：支撑 [04-功能树.md](04-功能树.md) 的角色矩阵；user 持 role_id、team、技能组。
- **sop_template / sop_version**：场景编码、节点规则、阈值（报价下限）、灰度比例、生效区间（§4.5）。
- **playbook / playbook_version**：话术正文、适用等级/渠道、RAG 索引状态、生效区间（§4.4）。
- **channel / assign_rule / stage_config**：渠道字典、分配规则（区域/负载/技能）、商机阶段定义。

## logs 库 · 要点（只追加，独立归档）
- **op_log**：谁/何时/对什么做了什么（人工操作审计）。
- **api_log**：开放 API 调用（requestId 幂等、耗时、结果）。
- **hermes_touch_log**：Hermes 触达记录，含 **trace_id**、SOP 节点、是否命中权限挂起（§4.3 回写）。
- **stage_change_log**：商机阶段变更轨迹。

## stats 库 · 要点（派生只读，可重算）
- **funnel_daily**：线索→跟进→商机→报价→赢单 各阶段日量与转化率（§4.7 漏斗）。
- **sales_perf_daily**：按人/组/渠道的跟进量、成交额。
- **playbook_hit_daily**：话术检索命中率、高意向未转化清单来源。

## 生命周期与保留
| 库 | 保留策略 |
|----|----------|
| cfg | 永久 + 版本留痕（sop/playbook 历史版本不删） |
| data | 永久；关闭/无效线索 1 年后归档 |
| logs | 热数据 90 天，之后冷归档；api_log 可压缩 |
| stats | 明细 13 个月，月级汇总永久；可由 data+logs 重算 |

## 生产级自检
- 每个实体**归属唯一库且有依据**（见上表“选择依据”）；跨库引用均 id-only（customer_id / owner_user_id / *_id）。
- PII：lead.phone、customer_id 关联信息为敏感字段，展示脱敏、访问审计（→logs.op_log）。
