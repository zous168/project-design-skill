# CRM 接口设计（清单大纲 + 接口定义 + OpenAPI/Swagger）

> **风格统一为 REST**(`/api/v1`):资源化复数 URL、标准动词 GET/POST/PUT/PATCH/DELETE + 正确状态码(201/202/204/4xx/5xx);非 CRUD 用子资源/动词路径(如 `POST /opportunities/{id}/close`);事件/队列接口归 [07 异步处理](07-异步处理.md)。下表清单即按此约定。

> 试跑产出（对应 skill 文档 **06 — Interface / API Design**）。先给**全量接口清单大纲**（与 [04-功能树.md](04-功能树.md) 操作一一对应），再给关键接口定义，最后说明 **OpenAPI/Swagger** 启用方式。风格：REST，`/api/v1`，鉴权 = 工作台 JWT / 开放 API 走 API Key（Hermes）。

## 2. 关键接口定义（示例）

### POST `/opportunities/{id}/special-price` — 申请/审批特价（§4.2 折扣权限）
- **入参**：`{ discount: number, reason: string }`
- **出参**：`{ approvalId, status: "pending|approved|rejected" }`
- **规则**：`discount < 0.9` 触发主管审批（写 data.special_price_approval）；申请 R1，审批 R2。
- **状态码**：200 / 400(折扣非法) / 403(越权) / 409(已有进行中审批)
- **幂等**：否（创建审批）；副作用：写 op_log。

### POST `/api/v1/leads` — Hermes 建线索（幂等）
- **入参**：`{ phone, channelUid, sourceChannel, campaign, customerId? }` + Header `X-Request-Id`
- **出参**：`{ leadId, deduped: boolean }`
- **规则**：按手机号/渠道 ID 查重，命中返回原 `leadId` 且 `deduped=true`（不新建）。
- **鉴权**：API Key（Hermes 服务账号 R4）；**幂等**：`X-Request-Id`。

### GET `/api/v1/sop?code=` — SOP 查询
- **出参**：`{ code, version, nextNode, interval, quoteFloor, escalateAfter, template }`（见 SOP 原型返回示例）。

## 3. 统一错误模型
```json
{ "code": "OPP_DISCOUNT_TOO_LOW", "message": "折扣低于权限下限，需主管审批", "traceId": "…" }
```
- HTTP 语义码 + 业务 `code`；4xx 客户端、5xx 服务端；写操作带 `X-Request-Id` 幂等；429 限流（开放 API）。

## 4. OpenAPI / Swagger（启用）
- 后端 NestJS 用 **`@nestjs/swagger`** 装饰器（`@ApiTags/@ApiOperation/@ApiResponse` + DTO `class-validator`）**自动生成 OpenAPI 3.0**。
- **Swagger UI** 挂载于 `/api/docs`（非生产环境开放；生产经鉴权）。
- OpenAPI JSON 导出至 `packages/contracts/openapi.json`，作为**契约源**：前端用它生成 TS 类型/请求客户端，避免字段漂移（呼应数据/接口一致性）。
- 版本：路径前缀 `/api/v1`；破坏性变更升 `v2`，旧版并行保留。
