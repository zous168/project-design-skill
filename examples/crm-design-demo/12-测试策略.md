# CRM 测试策略

> 全闭环产出 · 对应 skill 文档 **12**。策略,非用例/代码。上游:[01 PRD](01-需求规格.md)、[08 NFR](08-非功能性需求.md)。

## 1. 测试层级与范围
| 层级 | 范围 | 关注 |
|------|------|------|
| 单元 | service / 规则(查重 BR-1、折扣 BR-2) | 业务规则正确性 |
| 集成 | controller + PG + Redis;集成适配器(Hermes/客户全景/CSM)用契约/桩 | 接口契约、事务 |
| 异步/调度 | 队列消费、幂等、重试、DLQ、cron 防重跑 | 见 [07](07-异步处理.md) 可靠性语义 |
| E2E | 关键路径:进线→分配→跟进→赢单回写 | 跨模块闭环 |
| 性能 | 查重/检索 P95≤200ms @1k RPS | 见 [08 NFR](08-非功能性需求.md) |
| 安全 | 鉴权、越权(折扣绕过 RK-4)、脱敏、审计 | — |
| UI | 工作台/后台关键页 + 三态 + 三档响应式 + 主题/双语切换 | 浏览器点验 + Rubric D pre-flight |
| UAT | 坐席/主管/运营 角色场景 | 验收标准(01 US) |

## 2. 方法与环境
- 环境:dev / staging(私有化镜像)/ 仿真数据;外部域用契约桩(Pact 风格)。
- 测试数据:脱敏样本 + 工厂构造;PII 不入测试库明文。

## 3. 质量门 / DoD
- 单元+集成全绿、关键 E2E 通过、性能/安全门达标方可发布;覆盖率门按模块约定。

## 4. 需求可追溯(FR/NFR → 验证)
| 项 | 验证 |
|----|------|
| FR-L2 查重 | 单元(同号重复)+ 集成(返回原 id) |
| FR-O3 赢单回写 | 异步集成(三回写均产生记录)+ E2E |
| FR-O4 折扣权限 | 单元(阈值)+ 安全(越权拦截) |
| FR-S3 SOP 查询 | 集成(返回节点/权限/模板) |
| NFR 性能/i18n/主题/响应式 | 压测 / 切换验证 / 多视口 |

## 5. 工具类别与自动化
单元(Jest)· 集成(Supertest + testcontainers PG/Redis)· E2E(Playwright)· 性能(k6)· UI pre-flight(Rubric D / taste)。CI 跑单元+集成+关键 E2E。

---
### 审核与优化
- 每个 FR/NFR 至少一种验证;门客观;性能/安全/异步在 NFR 要求处均覆盖。状态 → **optimized**(doc_version 1)。
